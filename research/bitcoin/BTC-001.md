# BTC-001: Price Structure Research

**Phase:** 2 (Bitcoin — First Phase)
**Status:** Not started
**Objective:** Determine whether Bitcoin contains robust alpha using the same validation standards as Gold research.

---

## Research Question

Does Bitcoin (BTC-USD) exhibit statistically robust, tradeable price patterns when tested under the same rigorous validation framework applied to Gold (XAU/USD)?

## Dataset

| Source | Symbol | Coverage | Frequency |
|--------|--------|----------|-----------|
| Yahoo Finance | BTC-USD | ~2014–2026 | Daily OHLCV |

### Data Audit (Pre-Trade)

- Check completeness (missing values, gaps)
- Detect outliers and classify known events
- Test distributional assumptions (normality, stationarity)

## Validation Protocol

Every candidate signal must survive all applicable tests:

| Test | Description | Standard |
|------|-------------|----------|
| T1 | Basic screening | p<0.05, SR>1.0, PF>1.30, N>100 |
| T2 | Walk-forward | 3/4 temporal periods positive |
| T3 | Out-of-sample | Test SR>0, PF>1.0, decay<30% |
| T4 | Multiple testing | BH FDR q<0.05 |
| T5 | Monte Carlo | p<0.05 (5,000 permutations) |
| T6 | Drift neutralization | Alpha > 0 vs buy-and-hold |
| T7 | Directional consistency | Spearman p<0.05 (quintile monotonicity) |

## Signal Categories to Test

### 1. Return Distribution & Basic Statistics
- Summary statistics, normality tests
- Serial correlation (autocorrelation structure)
- Volatility estimation (realized, GARCH persistence)

### 2. Mean Reversion
- Z-score mean reversion (multiple windows/thresholds)
- Portfolio rebalancing effect

### 3. Trend Persistence
- Conditional probability of streak continuation
- Momentum signals (lookback periods)

### 4. Volatility Clustering
- ATR-based signals
- Regime-dependent return forecasts

### 5. Calendar Effects
- Day of week
- Month of year
- Halving cycle phase

### 6. Macro Event Impact
- FOMC days (if applicable)
- CPI releases
- Bitcoin-specific events (halving, ETF approvals)

## Success Criteria

Bitcoin research BTC-001 is considered complete when:

1. All signal categories have been tested
2. Full validation pipeline (T1–T7) applied to every candidate
3. Results documented in `reports/bitcoin/BTC-001_Report.md`
4. Framework modules from `research/gold/scripts/` reused where applicable

If no candidate survives all tests, Bitcoin — like Gold — is classified as **efficient under free data**.

## Dependencies

- `framework/statistics/` — summary stats, normality tests
- `framework/validation/` — T1 screening
- `framework/monte_carlo/` — permutation testing
- `framework/walk_forward/` — temporal validation
- `framework/drift_neutralization/` — alpha benchmarking
- `research/gold/scripts/` — parameterized Category B scripts

## Risk Factors

1. **Shorter history** (~4,000 daily bars vs ~6,466 for gold)
2. **Higher volatility** (~60-80% annualized vs ~18% for gold)
3. **Regime instability** — Bitcoin's market structure has changed dramatically (retail → institutional → ETF era)
4. **Weekend trading** — 24/7/365 market. Different session dynamics than traditional assets.
